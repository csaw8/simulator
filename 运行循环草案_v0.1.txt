运行循环草案 v0.1

一、文档目标

本文档用于定义项目初版的运行循环，说明世界如何在无人观察时持续运行、在关键节点自动细化、在少量 AI 参与下产出连续事件流，并以文本形式流式展示给观察者。

本文档以《世界模型草案_v0.1.txt》为前置基础。


二、初版运行循环的核心目标

| 目标 | 说明 |
|---|---|
| 世界持续运行 | 玩家不操作时，文明、地区与组织依然演化 |
| 宏观先行 | 大范围变化先由本地规则推进 |
| AI 节制介入 | 只在关键人物、关键时刻、关键观察中调用 |
| 事件连续可见 | 玩家可以边运行边看到世界事件逐步出现 |
| 节奏自动切换 | 平稳期粗推进，危机期细推进 |
| 成本可控 | 避免所有人物高频调用 AI |


三、运行循环总览

初版运行可视为一个重复执行的多阶段循环。

| 阶段序号 | 阶段名称 | 主要职责 | 是否调用 AI |
|---|---|---|---|
| 1 | 时间推进准备 | 决定本轮时间粒度与预算 | 否 |
| 2 | 宏观状态推进 | 推进文明、地区、组织的结构状态 | 否 |
| 3 | 压力与阈值检测 | 判断是否进入危机或阶段变化 | 否 |
| 4 | 候选角色筛选 | 找出本轮值得唤醒的人物 | 否 |
| 5 | 人物意图生成 | 让少量人物生成结构化意图 | 是 |
| 6 | 行动解析与冲突裁定 | 将人物意图与世界状态转成结果 | 否 |
| 7 | 基础事件生成 | 记录结构化事件流 | 否 |
| 8 | 叙事摘要生成 | 为高价值事件生成短文本 | 低频是 |
| 9 | 流式展示输出 | 将事件流推给观察界面 | 否 |
| 10 | 存储与缓存更新 | 更新状态、工作记忆与短期缓存 | 否 |


四、一次完整运行循环的主逻辑

初版推荐主循环如下：

1. 根据当前世界态势决定本轮推进粒度。
2. 推进宏观状态，包括文明、地区、派系和阶段压力。
3. 生成结构性变化候选，如资源紧张、政局恶化、边疆冲突、科技失控等。
4. 根据结构性变化筛选需唤醒的主角与活跃人物。
5. 仅对高优先级角色调用 AI，生成结构化意图。
6. 将结构状态与角色意图交给裁定层，得出行动结果。
7. 生成结构化基础事件，追加到事件流。
8. 从事件流中挑选少量高价值事件，调用史官摘要生成短文本。
9. 将基础事件和叙事事件逐步推送到文本观察界面。
10. 更新世界状态、角色记忆摘要、观察缓存与工作记忆。


五、阶段一：时间推进准备

时间推进不是固定 tick，而是“默认粗粒度 + 条件触发细化”。

建议输入：

| 输入 | 来源 |
|---|---|
| 当前全局时间 | 世界状态 |
| 当前文明阶段 | 文明模型 |
| 当前地区紧张度 | 地区模型 |
| 未处理高危事件 | 事件流 |
| 当前观察焦点 | 玩家界面状态 |

建议输出：

| 输出 | 含义 |
|---|---|
| 本轮时间粒度 | 周 / 日 / 半日 / 小时 |
| 本轮 AI 预算 | 主角预算、活跃人物预算、史官预算 |
| 本轮细化标记 | 是否进入关键时刻 |

粒度建议规则：

| 条件 | 推荐粒度 |
|---|---|
| 世界整体平稳 | 周 |
| 某地区进入高紧张 | 日 |
| 战争、政变、灾害、事故升级 | 半日 |
| 关键角色直接冲突 | 半日或小时 |
| 玩家聚焦危机节点 | 临时细化 |

说明：

1. 初版默认时间单位为“周”。
2. 粒度细化不是美术效果，而是为了保证事件链连续。
3. 细化粒度只持续到关键事件被处理完毕。


六、阶段二：宏观状态推进

这一阶段完全不依赖 AI，由本地规则推进“模糊客观状态”。

推进对象：

| 对象 | 推进内容 |
|---|---|
| 文明 | 阶段压力、凝聚力、扩张压力、正统性、技术轮廓变化 |
| 地区 | 安定度、繁荣度、稀缺度、基础设施负荷、政治紧张度 |
| 派系 | 影响力、控制区、敌对度、内部凝聚 |
| 非凡物 | 状态变化触发条件检测，例如失控、争夺、激活趋势 |

推进方式建议：

1. 使用标签和趋势值，而非精确数值。
2. 变化以“低、中、高”和“上升、持平、下降”为主。
3. 结构推进优先体现长期惯性，而不是每轮大幅震荡。

这一阶段的目标不是制造戏剧，而是给戏剧提供真实压力背景。


七、阶段三：压力与阈值检测

宏观推进之后，系统需要检测哪些变化已经足够显著，值得转化为事件或角色唤醒条件。

可检测对象包括：

| 类别 | 示例 |
|---|---|
| 资源压力阈值 | 充裕 -> 紧张，紧张 -> 饥荒 |
| 治理阈值 | 稳定 -> 摇摆，摇摆 -> 失序 |
| 冲突阈值 | 对峙 -> 局部冲突 -> 战争升级 |
| 阶段阈值 | 联邦/帝国期 -> 裂解期 |
| 技术阈值 | 研究阶段 -> 试验阶段 -> 失控阶段 |
| 信念阈值 | 异端扩散 -> 群众狂热 |

输出结果建议：

| 输出 | 用途 |
|---|---|
| 结构性事件候选 | 直接进入事件裁定候选池 |
| 角色唤醒标记 | 用于下一阶段候选人物筛选 |
| 粒度升级标记 | 触发后续时间细化 |


八、阶段四：候选角色筛选

这一阶段是成本控制的关键。不是所有人物都进入 AI 调用。

候选池来源：

| 来源 | 说明 |
|---|---|
| 主角人物池 | 默认优先级最高 |
| 活跃人物池 | 按条件进入 |
| 观察痕迹人物池 | 玩家近期关注过的对象 |
| 临近升格人物池 | 从配角中临时析出的潜在重要人物 |

建议筛选逻辑：

1. 先构建候选列表。
2. 为每个角色计算唤醒优先级。
3. 按分数排序。
4. 按本轮 AI 预算截断。

优先级组成建议：

| 因子 | 作用 |
|---|---|
| 角色级别 | 主角 > 活跃人物 > 配角 |
| 地区紧张度 | 危机地区优先 |
| 派系冲突度 | 所属组织进入冲突时优先 |
| 关系变动 | 关键人物出事会拉高相关者 |
| 近期目标紧迫度 | 即将成功或失败的人优先 |
| 玩家关注度 | 最近被观察过的人优先 |
| 叙事潜力 | 有连续故事价值的人优先 |

建议结果：

| 分数区间 | 处理方式 |
|---|---|
| 高分 | 本轮必唤醒 |
| 中分 | 有预算才唤醒 |
| 低分 | 本轮休眠，沿用旧意图或群体逻辑 |


九、阶段五：人物意图生成

只有通过筛选的人物才进入 AI 调用。

AI 在这一阶段不负责“决定世界真相”，只负责输出人物层面的结构化意图。

建议输入内容：

| 内容 | 说明 |
|---|---|
| 世界母风格锚点 | 现实未来科技发展风格 |
| 角色卡 | 身份、欲望、恐惧、关系、最近目标 |
| 局部世界摘要 | 只给与角色相关的文明/地区/派系信息 |
| 近期相关事件 | 最近 3-8 条关键事件 |
| 输出格式约束 | 结构化 JSON 或等价结构 |

建议输出字段：

| 字段 | 含义 |
|---|---|
| character_id | 角色 ID |
| intent_type | 意图类型 |
| target | 目标对象 |
| goal | 本轮想达成的目的 |
| motivation | 为什么这么做 |
| risk_tolerance | 风险承受倾向 |
| urgency | 紧迫度 |
| proposed_action | 建议行动 |
| tone | 情绪或态度 |

说明：

1. 输出必须短而结构化。
2. 不允许在此阶段让模型自由编造全新世界事实。
3. 人物意图只是提案，不是最终现实。


十、阶段六：行动解析与冲突裁定

该阶段由本地规则处理，是世界客观性的最后防线。

裁定输入：

| 输入 | 来源 |
|---|---|
| 当前世界状态 | 宏观状态、地区状态、派系状态 |
| 角色意图列表 | 阶段五输出 |
| 非凡物状态 | Relic 模型 |
| 当前时间粒度 | 阶段一输出 |

裁定任务：

| 任务 | 说明 |
|---|---|
| 可行性校验 | 角色是否有条件执行该动作 |
| 冲突处理 | 多角色目标冲突如何竞争 |
| 环境修正 | 资源不足、治安恶化、地区封锁等会改变结果 |
| 风险落地 | 行动是否引发伤亡、声望变化、组织对立 |
| 后果传播 | 结果如何影响地区、派系、文明状态 |

裁定原则：

1. AI 提供人物意图，系统裁定结果。
2. 任何重大结果都必须能够回落到世界状态字段。
3. 裁定优先维护因果一致性，而不是戏剧性最大化。


十一、阶段七：基础事件生成

行动裁定完成后，系统应生成结构化基础事件。

基础事件来源包括：

| 来源 | 示例 |
|---|---|
| 宏观阈值变化 | 边疆失序、补给中断、城市断电 |
| 角色行为结果 | 谈判破裂、叛逃、窃取、联盟、袭击 |
| 非凡物状态变化 | 巨构重启、禁物泄露、圣物失窃 |
| 阶段转换 | 文明裂解、区域独立、技术跃迁 |

基础事件特点：

| 特点 | 说明 |
|---|---|
| 结构化 | 优先字段化记录 |
| 短句化 | 界面可以直接显示 |
| 可追溯 | 保留因果标签 |
| 可聚合 | 便于后续摘要 |

建议基础事件展示例子：

1. 北轨港区安定度由“摇摆”跌入“失序”。
2. 第七物流工团拒绝向轨道升降站输送储能芯块。
3. 主角人物林逾白试图借工潮逼迫议会让步，但遭到安保部门提前截获。


十二、阶段八：叙事摘要生成

不是每个基础事件都需要 AI 润色。

该阶段只处理高价值事件：

| 适合生成叙事摘要的事件 | 示例 |
|---|---|
| 连续事件链开始成形 | 地区骚乱从物资争夺演变为制度危机 |
| 主角引发重大变化 | 一名关键人物的决定改变地区局势 |
| 文明阶段变化 | 某文明由高技跃迁期进入裂解期 |
| 非凡物强烈介入 | 巨构复苏带来新的政治中心 |
| 玩家正在聚焦 | 需要更细腻的观察文本 |

输出建议：

| 输出类型 | 说明 |
|---|---|
| 短旁白 | 1-3 句，对事件链作简短史官化表达 |
| 局势摘要 | 对某地区/文明当前局面作一段压缩说明 |
| 角色转折记录 | 标记某人物走入历史舞台的瞬间 |

说明：

1. 叙事摘要是低频批量生成，不是逐条事件都生成。
2. 叙事摘要属于文本投影，不应反过来改写基础事件。


十三、阶段九：流式展示输出

你已明确希望“边运行边流式看到事件出现”，因此界面层应按流方式展示，而不是每轮只吐一段总结。

推荐输出分层：

| 输出层 | 内容 | 节奏 |
|---|---|---|
| 实时事件层 | 基础结构化事件 | 高频 |
| 旁白层 | 对关键事件的短摘要 | 中低频 |
| 世界概况层 | 文明、地区、阶段状态概览 | 低频或按需 |

推荐展示规则：

1. 基础事件先出现。
2. 关键节点稍后出现史官短评。
3. 当局势进入细粒度时间时，事件流密度自然提高。
4. 玩家观察某对象时，事件流可临时偏向该对象相关内容。


十四、阶段十：存储与缓存更新

运行一轮后，系统必须将结果写回各层存储。

建议更新目标：

| 更新目标 | 内容 |
|---|---|
| State | 文明、地区、派系、人物、非凡物的新状态 |
| Event Stream | 追加本轮基础事件与叙事事件 |
| Working Memory | 活跃角色近期目标、局部局势摘要、待处理线索 |
| Observation Trace | 玩家本轮看过的对象 |
| Flavor Cache | 观察时生成的临时细节 |

更新原则：

1. 客观状态先更新，再更新叙事文本。
2. 观察生成内容默认不进入正史。
3. 人物长期记忆不保存全部过程，只保留压缩摘要。


十五、平稳期与关键期的双模式循环

初版建议世界运行具备两种模式：

| 模式 | 适用情况 | 特征 |
|---|---|---|
| 平稳模式 | 世界无高危事件，地区局势总体稳定 | 周推进、低调用、低密度事件流 |
| 关键模式 | 某地区或角色链条进入高度紧张状态 | 日/小时推进、高密度事件流、更多角色唤醒 |

切换条件建议：

| 条件 | 结果 |
|---|---|
| 地区紧张度达到高位 | 进入关键模式 |
| 主角冲突直接发生 | 进入关键模式 |
| 重大非凡物被激活 | 进入关键模式 |
| 连续数轮未出高危事件 | 回到平稳模式 |


十六、观察者行为如何插入循环

初版玩家主要是观察者，因此玩家行为不应打断世界逻辑，而应作为附加请求插入。

观察行为插入点：

| 行为 | 插入时机 |
|---|---|
| 观察文明 | 可在任意轮后读取概况 |
| 观察地区 | 可在事件流推进间隙读取 |
| 观察人物 | 可在角色意图生成后或事件发生后读取 |
| 观察非凡物 | 可在事件摘要后读取 |

影响：

| 影响项 | 初版是否生效 |
|---|---|
| 改变世界状态 | 否 |
| 提高角色未来唤醒权重 | 是 |
| 生成风味细节缓存 | 是 |
| 产生新的客观事件 | 否 |


十七、运行循环中的 AI 预算建议

为了控制 DeepSeek API 成本，运行循环中必须显式维护预算。

建议预算类别：

| 预算类型 | 用途 |
|---|---|
| 主角预算 | 高频但总量受限的关键角色调用 |
| 活跃角色预算 | 条件唤醒人物调用 |
| 史官预算 | 批量摘要生成 |
| 观察预算 | 玩家手动观察时的补完文本 |

建议平衡档初版配置：

| 项目 | 建议值 |
|---|---|
| 主角调用 | 每周 4-6 次总量 |
| 活跃人物调用 | 每周 4-10 次总量 |
| 史官摘要调用 | 每周 2 次左右 |
| 观察补完 | 按需 |

说明：

1. 不是每个主角每周都一定调用。
2. 预算应根据世界是否进入关键模式动态增减。


十八、错误与异常处理建议

运行循环必须容忍以下问题：

| 问题 | 建议处理方式 |
|---|---|
| AI 调用失败 | 回退到旧意图或模板反应 |
| 输出结构不合法 | 丢弃并重试一次，仍失败则回退 |
| 某角色连续多轮未被唤醒 | 允许保持休眠，不强制补算 |
| 事件流过密 | 优先压缩基础事件，避免界面淹没 |
| 关键期持续过长 | 自动降采样并生成局势摘要 |

说明：

1. 世界不能因为一次模型失败而停摆。
2. 本地规则层必须能在 AI 不可用时继续运转。


十九、初版运行循环的最小闭环

如果需要最小可行原型，建议只做以下闭环：

| 步骤 | 最小实现 |
|---|---|
| 时间推进 | 默认按周推进 |
| 宏观推进 | 文明/地区状态变化 |
| 候选筛选 | 从主角和活跃人物中筛选 |
| 意图生成 | 少量角色调用 AI |
| 裁定 | 本地规则判定结果 |
| 事件流 | 结构化事件追加 |
| 摘要 | 每若干事件生成简短旁白 |
| 展示 | 文本流式输出 |

只要这条链能跑通，你的世界就已经“活起来了”。


二十、运行循环的核心判断顺序

若实现时出现冲突，应遵循以下优先级：

| 优先级 | 判断准则 |
|---|---|
| 1 | 世界必须能在无人观察时继续运行 |
| 2 | 宏观变化必须先于个体叙事出现 |
| 3 | AI 只负责人物意图与叙事投影，不直接定义世界真相 |
| 4 | 事件流是主记录，摘要文本是附加层 |
| 5 | 预算优先保证主角与关键事件，不平均分配给所有角色 |
| 6 | 初版观察只读，不塑形 |
