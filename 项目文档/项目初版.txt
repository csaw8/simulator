一、 核心需求总结 (系统愿景)
你的目标是构建一个高自由度的数字培养皿 (Digital Terrarium)，具体包含以下四大核心特征：

双轨解耦运行： 放弃 AI 全盘接管，改用确定性的本地代码引擎（快系统）负责物理、资源与状态演算，AI（慢系统）仅作为“旁白”和“命名者”异步介入。

薛定谔的细节 (延迟加载)： 世界的微观细节（如普通平民的穿着、量产武器的来历）在未被观察时不占用运算资源。仅在玩家点击“聚焦观察”时，调用大模型结合上下文即兴生成。

分级持久化 (Tiered Persistence)： * 史诗层： 重大历史事件、传奇人物、神器的描述文本由 AI 生成后，永久写入本地数据库成为“正史”。

风味层： 普通个体的即兴细节阅后即焚，通过 LRU 缓存或哈希种子（Hash Seed Determinism）在短期内维持一致性，不占用永久存储。

动态升格机制 (Ascension)： 底层的普通实体一旦在演算中触发了极低概率的重大事件（如小兵斩杀巨龙），系统将自动为其打上 Is_Unique 标签，触发 AI 生成专属史诗背景，并从“风味层”永久跃迁至“史诗层”。

二、 项目核心特点与设计哲学
1. 叙事基调：带有温度的“叹息史官”

身份设定： 玩家/系统扮演一个跨越无尽岁月的“高维观察者”。

文本风格： 拒绝纯粹无机质的代码日志，大模型将以“克制的叹息”和宿命感来记录文明的兴衰。底层是冰冷的物理演算，表层是具有人文温度的史诗。

2. 运行节奏：深度放置与自适应时间轴

时间控制： 具备极高自由度的时间流速调节。系统可以在后台以高倍率自动推演，类似那些拥有极深系统机制的展开式放置游戏（Unfolding Incremental Games，如 Kittens Game 或 Evolve Idle），在脱机状态下依然进行着严密的资源迭代和文明演化。

主观干预： 玩家只有在想扮演“第四天灾”投放灾变或恩赐时才产生交互，其余时间系统依靠底层规则与 AI 阈值判定自行运转。

3. 世界生命周期：灭绝、重启与遗产继承

残酷底色： 允许文明在恶劣的物理规则下彻底灭绝。

Meta 机制： 演化可以有终局，世界毁灭后触发“重启”。前代文明的科技巅峰或史诗造物将化作“高维数据碎片”或远古遗迹，继承到下一个纪元中。

维度飞升： 极小概率下，文明推演突破当前底层物理规则引擎的上限，触发平滑切换，进入更高级别的维度模拟。

4. 终极特权：时光回溯与平行宇宙残像

存档回退： 玩家作为高维法则，可以强行拨回时间指针，拯救注定灭亡的文明。

废弃时间线： 被剪断的毁灭时间线不会被物理删除，而是作为“版本分支”静静封存在数据库中，成为未来可能引发系统异变（Glitch）或原住民“既视感”幻觉的暗线彩蛋。

三、技术架构与实现路线
为了支撑上述宏大且复杂的特性，系统采用“双轨运行、完全解耦”的现代工业级架构。这能完美隔离底层数值的稳定性与顶层 AI 生成的波动性。

1. 快系统：底层物理演算内核 (C++ / Godot 4)
这是培养皿的“心脏”，负责处理成千上万个实体的每秒高频推演，绝对不依赖大模型。

技术栈选型： 利用 Godot 4 搭配 C++ GDExtension 来构建无 UI 的高性能沙盒内核是非常契合的选择。C++ 负责极速的内存级演算，Godot 提供便利的节点管理、可视化 UI 绑定和事件分发。

ECS 架构： 采用实体-组件-系统（Entity-Component-System）。世界由海量携带属性标签（如人口、理智、科技值）的 JSON/内存对象构成。

事件总线： 当底层数值触发特定阈值（如：跨级击杀、资源枯竭）时，抛出结构化数据事件，供外层捕获。

2. 慢系统：异步 AI 渲染服务 (Python / Gemini API)
这是培养皿的“灵魂与喉舌”，仅在需要文本输出或复杂决策时被事件唤醒。

服务端： 搭建独立的 Python 异步后端（如 FastAPI/Celery），监听来自 C++ 内核的事件。

大模型接入： 对接 Gemini API 等大语言模型，利用多 Agent 架构（史官 Agent、细节渲染 Agent、命名 Agent）处理不同类型的文本生成任务。

3. 动态数据与记忆管理机制 (核心难点突破)
薛定谔的细节（延迟加载）： 世界微观细节（平民性格、量产武器外观）仅在玩家点击“聚焦观察”时，通过哈希种子（Hash Seed）即兴调用 AI 生成。生成的文本存入 LRU 内存缓存，阅后即焚，不落盘。

动态升格（Ascension）： 普通实体一旦触发底层低概率重大事件，其组件将被打上 Is_Unique 标签。AI 为其生成专属史诗背景，从此跃迁为重要实体。

分级持久化与向量库： 升格后的史诗文本、重大历史事件日志，将带有时间戳永久存入本地数据库和向量数据库（Vector DB）。这既构成了主时间线的“正史”，也妥善保存了回溯操作产生的“废弃时间线”。
