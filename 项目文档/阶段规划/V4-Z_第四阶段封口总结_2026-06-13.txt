| 项 | 结论 |

第四阶段的目标不是把项目变成通用多题材模拟器，而是把当前默认题材的文风、命名、prompt、descriptor 边界集中管理，并为第五阶段半开放结构生成提供清晰边界。这个目标已经达到。

第四阶段已经提供了第五阶段需要的边界：

```text
style profile 决定题材边界
narrative lexicon 决定显示名与观察标题
descriptor lexicon 决定 AI 可选隐标签空间
descriptor profile 提供可保存、可查询的描述档案
AI proposal audit 提供可追踪的写入路径
```

九、第五阶段入口建议

| 顺序 | 批次 | 内容 | 说明 |
|---:|---|---|---|
| 1 | V5-A | 半开放结构模板最小 schema | 定义 AI 可提议的新结构类型模板，但模板本身必须受控 |
| 2 | V5-B | StructureTemplateRegistry | 注册 allowed structure templates，不允许 AI 任意创建字段 |
| 3 | V5-C | AI structure proposal dry-run | AI 只提交候选结构模板实例，先不 apply |
| 4 | V5-D | validation / audit / apply | 通过 validator 后才写入世界状态 |
| 5 | V5-E | player/truth summary 最小接入 | 只接一类新结构，验证信息差 |
| 6 | V5-F | DescriptorProfile 联动 | 新结构必须声明可用 descriptor 字段和 approved pool |
| 7 | V5-G | SQLite / snapshot 扩展 | 新结构可保存、导出、审计 |
| 8 | V5-H | 小规模自由度扩宽 | 允许 AI 在模板内组合字段，不允许自由新增 tag |
| 9 | V5-I 或更后 | AI 自由新增 descriptor tag 试点 | 必须 proposal / validation / approval / audit |

十、第五阶段第一批建议

建议第五阶段先做：

```text
V5-A：半开放结构模板最小 schema
```

推荐原因：

| 原因 | 说明 |
|---|---|
| 衔接明确 | 第四阶段已经完成 style/lexicon/descriptor 边界 |
| 风险可控 | 先做 schema 和 registry，不直接大规模 apply |
| 对目标有效 | 用户想要“AI 生成结构与固定结构并列、结构独立、功能互相影响”，V5-A 正是入口 |
| 不污染现有模型 | 新结构必须通过模板和 validator，不直接改固定对象字段 |

十二、结论

第四阶段可以封口。

下一阶段不应继续做“更多文案迁移”，而应开始第五阶段的半开放结构生成：

```text
固定结构继续作为主干
AI 结构作为并列扩展层
结构独立
功能互相影响
所有写入走 proposal -> validation -> audit -> apply
```
