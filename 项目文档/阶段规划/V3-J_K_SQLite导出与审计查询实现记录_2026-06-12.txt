二、设计边界

| 边界 | 说明 |
|---|---|
| JSON 快照仍是权威 | SQLite 只是导出查询层 |
| 不替代 WorldState | 不从 SQLite 反向恢复世界 |
| 不为每类模板建新表 | 半开放结构阶段前保持通用 payload_json |
| 不让 AI 直接写库 | AI 仍走 proposal -> validation -> audit -> apply |

三、表结构

| 表 | 用途 |
|---|---|
| `events` | 查询事件时间线、类型、可见性、payload |
| `relations` | 查询对象关系和强度 |
| `dynamic_structures` | 查询 DynamicStructure 当前状态 |
| `emergent_presences` | 查询 EmergentPresence 当前状态 |
| `ai_proposal_audits` | 查询 AI proposal 质量与错误 |

五、结论

V3-J / V3-K 已完成最小查询层。后续如果需要更复杂的 `db audits`、`db dynamics`、`db emergents` 命令，可以在现有 SQLite 表上继续加只读查询，不需要改变世界权威存储。
